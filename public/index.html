Community Data Scraper Service API

This service provides community events data from Google Calendar feeds with personalized ranking.

Available Endpoints

GET /api/status
Get the health status of the service.
GET /api/feed
Get community events feed with optional personalization and filtering.
Query Parameters:
start: Start date (ISO format, defaults to today)
days: Number of days to fetch (1-31, defaults to 1)
tags: Comma-separated list of tags to filter by (max 10)
userId: User ID for personalized ranking
pageSize: Number of events per page (1-50, defaults to 20)
pageToken: Token for pagination
GET /api/tag-proposals
Get top tag proposals for autocomplete.
Query Parameters:
limit: Number of proposals to return (1-100, defaults to 20)
POST /api/interactions
Record a user interaction with content.
POST /api/interactions/batch
Record multiple user interactions (max 100).

Mock Data Endpoints

GET /api/mock/feed
Get mock content feed with personas or hybrid mode.
POST /api/mock/interactions
Record mock interactions (no-op for personas, real for hybrid).

Example Usage

// Get events for the next week
GET /api/feed?start=2025-09-29&days=7

// Get personalized feed for a user
GET /api/feed?userId=user123&days=3

// Filter by tags
GET /api/feed?tags=yoga,fitness&days=7

// Get top tag proposals
GET /api/tag-proposals?limit=50

// Mock persona feed
GET /api/mock/feed?userId=mock-foodie

Data Updates

Community events are automatically synced from Google Calendar every 30 minutes.

Events are classified with LLM-generated tags and embeddings for personalized ranking.
